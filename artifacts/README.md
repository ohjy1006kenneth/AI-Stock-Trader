# artifacts

Artifact schemas and placeholder directories.

Large generated model bundles and deployment build outputs are intentionally not committed.
Keep only lightweight structure, schemas, manifests, and small reference files in git.

Typical subfolders:
- `bundles/` — packaged model or deployment bundles
- `deployments/` — deployment outputs and release-ready artifacts
- `reports/` — generated run reports and validation summaries
- `logs/` — deterministic runtime logs
